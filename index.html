Trading Probability Dashboard

Instructions:

This dashboard analyzes trading state transitions. Below you'll find the probabilities for different state transitions based on historical data.

To use this dashboard with your data:

  1. Make sure your Google Sheet is publicly accessible
  2. The data should have columns for Date, Day, FROM_STATE, TO_STATE, and transition information
  3. Use the calculations below as a template for your analysis

PREVIOUS DAY (FROM_STATE)

FROM STATE

Select a FROM_STATE to analyze:

  • 000 - All bits are 0
  • 001 - ADR=0, ODR=0, RDR=1
  • 010 - ADR=0, ODR=1, RDR=0
  • 011 - ADR=0, ODR=1, RDR=1
  • 100 - ADR=1, ODR=0, RDR=0
  • 101 - ADR=1, ODR=0, RDR=1
  • 110 - ADR=1, ODR=1, RDR=0
  • 111 - All bits are 1

CURRENT DAY (TO_STATE)

TO STATE

The TO_STATE represents the state that occurs after a given FROM_STATE.

Each state has three digits representing:

  • First digit (ADR): Asian/After-hours session
  • Second digit (ODR): Overnight/London session
  • Third digit (RDR): Regular/New York session

Progressive Prediction

Step 3: Predict RDR (Third Digit)

If ADR = 0 and ODR = 0, what is the probability of RDR being 0 or 1?

RDR = 0 66.7% (2 out of 3)

How to Use This Data

  1. Identify your current FROM_STATE
  2. Look up the transition probabilities for that state
  3. Use the progressive prediction to estimate each digit of the TO_STATE
  4. The higher probability outcomes are more likely to occur

Note: These probabilities are based on historical data and do not guarantee future results.
